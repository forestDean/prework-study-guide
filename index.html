HTML

  • <!DOCTYPE html> - declaration tells the browser what document type it should expect. It is case-insensitive.
  • <html> - element represents the root/start of our HTML document and can contain language attributes.
  • <head> - element contains information about the webpage in metadata tags/elements.
  • <body> - element represents the visible content shown to the user.
  • <title> - element defines the title for a webpage. It is required and important for SEO.
  • <style> - element defines the CSS styles associated with a website.
  • <base> - element defines the base URL for a webpage.
  • <link> - element connects an external resource to the HTML document.
  • <style> - element defines metadata such as the character set, description, keywords, author, and viewport.
  • <script> - element embeds client-side scripts in the HTML document.
  • <div> - element is non-semantic and used to create a division in the page.
  • <span> - element is non-semantic and used to create an inline division.
  • <h1> <h2> <h3> <h4> <h5> <h6> - elements represent the level of heading a text block represents.
  • <p> - element represents a paragraph or block of text.
  • <ul> - element represents an unordered list.
  • <ul> - element represents an unordered list.
  • <ol> - element represents an ordered list.
  • <li> - element represents a list item.
  • <img> - element contains information about images on the webpage. It contains various attributes.
  • <br> - element creates a line break of empty space.
  • <a> - element creates links to the same webpage or other webpages. The href attribute points to the URL for the link.
  • rel - attribute specifies the relationship between the current document and the linked document resource..
  • href - attribute specifies the location URL of the external resource using a relative/absolute path.

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • The selector defines the element/attributes to which the rules/declarations will apply.
  • Declarations contain two components: the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want around the outside of an element.
  • A padding property adds space around the content inside an element.
  • Use a special wild card symbol * to apply a rule to all the elements visible on the page.
  • Combined two selectors into a single CSS rule with a comma.
  • The block value assigns each 'image' to start on a new line.
  • Class attributes allow us to share a CSS rule to any element by assigning the rule to a class attribute with a class selector.

Git

  • git status - checks what branch we are currently on.
  • git pull origin main - pulls down the latest main branch.
  • git checkout -b branch-name - creates a new branch and switches to it.
  • git add -A - stages all files in the working branch.
  • git commit -m "short message" - commits staged files to our repository, with a message.
  • git push origin branch-name - pushes 'branch-name' changes to the GitHub repository.
  • cd directory-name - changes directory.
  • touch file-name - creates a file.

JavaScript

  • var - variable is a named container that allows us to store data in our code.
  • = - operator is used to assign the specific data that we want stored in our variable.
  • The javascript file is linked within the <body> element.
  • Control flow is the order in which a computer executes code in a script.
  • To signify that the variable value is a string, enclose it in single or double quote marks.
  • To signify that the variable value is an array, enclose it in square brackets.
  • In an array strings need to be contained within quotation marks, other values do not.
  • Everything in JavaScript is an object and can be stored in a variable.
  • You can write comments in JavaScript just as you can in CSS.
  • Conditionals are code structures used to test if an expression returns true or not.
  • Functions define a body of code that executes when you call the function name in your code.
  • Functions often take data arguments to do their job. They go inside the parentheses, separated by commas.
  • function () is an anonymous function and doesn't have a name.
  • function () can be written as an arrow function using () =>
  • function () is an anonymous function and doesn't have a name.
  • A value is falsy if it is: 0, -0, On, "", null, undefined, NaN, or a Boolean data type value false.
  • If a value is not falsy it is truthy.
  • A function is declared with a unique name followed by parentheses.
  • The function declaration is followed by the code block in curly brackets.
  • To execute a function we must call it in our code.
